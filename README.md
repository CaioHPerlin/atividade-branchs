✔️ - Cada funcionalidade deve ter no mínimo 2 commits por branch
✔️ - O nome do branch deve seguir o padrão feature/nome-da-funcionalidade
❌ - Fazer 2 merge direto no master
✔️ - Fazer 3 pull-request no master
✔️ - Adicionar um colega como colaborador

❌ - Cada bugfix deve ser feito em um branch
❌ - Ao final fazer o pull-request de cada funcionalidade no branch principal e colocar um colega como revisor do código
❌ - O nome do branch deve seguir o padrão bugfix/nome-da-funcionalidade
❌ - Adicionar 2 funcionalidades por colaborador, com 2 commits cada
❌ - No meio do desenvolvimento de cada funcionalidade (após 1 commit), simular que apareceu um bug urgente para correção, e começar imediatamente o bugfix.
❌ - Depois voltar e terminar a funcionalidade
❌ - Cada tarefa deve ser feita em branch separado
❌ - Fazer o merge de cada atividade no master
